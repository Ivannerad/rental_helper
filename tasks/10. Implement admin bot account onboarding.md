# Why
Admins need a controlled flow to register a Telegram account and complete Telethon authorization.

# Context
- Lives in `src/admin_bot/` and shared account/session persistence modules.
- Current behavior: there is no way to add managed accounts.
- Expected behavior: the admin bot guides account creation, captures required credentials, and stores the resulting session.

# Constraints
- Do not expose raw Telegram credentials in chat history beyond what is necessary.
- Persist onboarding progress so interrupted flows can resume safely.
- The final result must be compatible with the Telethon worker runtime.

# Acceptance Criteria
- Admin commands support creating a managed account record and completing authorization.
- Session data is stored securely and linked to the managed account.
- The onboarding flow reports clear success and failure states.

# Validation
- Run tests for onboarding state transitions.
- Manually complete account authorization for a test Telegram account.

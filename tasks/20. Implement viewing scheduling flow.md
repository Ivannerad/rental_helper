# Why
Once a user accepts a listing, the system must capture a proposed viewing date and time and store it reliably.

# Context
- Lives in `src/chatbot_service/` appointment modules and admin schedule views.
- Current behavior: there is no appointment creation flow.
- Expected behavior: the chatbot asks for viewing time details, stores a pending appointment, and makes it visible to admins.

# Constraints
- Appointment records must link account, user, conversation, and listing.
- Partial scheduling information should remain explicit rather than inferred silently.
- The flow must work in Russian and support clarification prompts.

# Acceptance Criteria
- The chatbot can collect date and time details after listing acceptance.
- A `ViewingAppointment` record is created or updated in persistence.
- The appointment appears in admin schedule output.

# Validation
- Run unit and integration tests for appointment creation and retrieval.
- Manually confirm an agreed viewing appears in the admin bot schedule command.

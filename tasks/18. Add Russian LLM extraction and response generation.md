# Why
The assistant must understand tenant intent and reply naturally in Russian while keeping control logic deterministic.

# Context
- Lives in `src/chatbot_service/llm/` and prompt/config modules.
- Current behavior: no LLM integration exists.
- Expected behavior: the service calls an LLM to extract requirements and draft Russian replies, then maps that output into policy decisions.

# Constraints
- Do not let the LLM directly decide state transitions or broker actions.
- Prompting must target Russian-only user communication in v1.
- Handle provider errors and malformed outputs without losing conversation state.

# Acceptance Criteria
- A typed LLM client abstraction is implemented.
- Prompts and output parsing support requirement extraction, intent classification, and Russian response drafting.
- The service falls back safely when the model output is invalid or uncertain.

# Validation
- Run unit tests for output parsing and fallback handling.
- Manually execute sample Russian prompts against the configured provider.

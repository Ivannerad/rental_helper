# Why
The chatbot requires deterministic stage control to avoid inconsistent user flows.

# Context
- Lives in `src/chatbot_service/`.
- Current behavior: no conversation engine exists.
- Expected behavior: conversations move through `collecting`, `searching`, `viewing`, and `upsell` states according to explicit rules.

# Constraints
- State transitions must be deterministic and testable outside the LLM.
- Persist stage and conversation context after each handled message.
- Human handoff must be able to pause automation cleanly.

# Acceptance Criteria
- A state machine module defines allowed transitions and stored stage data.
- The service can resume a conversation from persisted state.
- Invalid or ambiguous transitions are handled safely.

# Validation
- Run unit tests for all stage transitions and pause/resume behavior.
- Manually replay sample conversations through each stage path.

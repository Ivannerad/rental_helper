# Why
The project needs reproducible install, lint, format, and test commands before feature work begins.

# Context
- Lives in root project configuration such as `pyproject.toml` and developer docs.
- Current behavior: no canonical tooling or commands are defined.
- Expected behavior: contributors can install dependencies and run checks the same way.

# Constraints
- Prefer a single project configuration file where practical.
- Keep commands compatible with a multi-service Python repository.
- Do not add file-rewriting automation that hides errors from contributors.

# Acceptance Criteria
- A Python project configuration file defines dependencies and dev dependencies.
- Canonical commands for install, lint, format, and test are documented.
- The repo can execute at least a smoke test command in a fresh environment.

# Validation
- Run install, lint, format-check, and test commands.
- Manually confirm the commands are documented in `README.md` and `AGENTS.md`.

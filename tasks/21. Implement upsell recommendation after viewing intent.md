# Why
The product requires one additional similar offer after a viewing is accepted or completed.

# Context
- Lives in `src/chatbot_service/` post-viewing recommendation logic.
- Current behavior: no upsell stage exists.
- Expected behavior: after the viewing stage, the chatbot selects one similar listing in the same area and offers it in Russian.

# Constraints
- Offer only one upsell candidate per eligible stage entry in v1.
- The upsell must respect the same account and bound group inventory.
- Do not loop endlessly if no similar listing exists.

# Acceptance Criteria
- The state machine can enter and exit the `upsell` stage.
- A similar listing is selected using area and related constraints.
- If no similar listing exists, the conversation exits gracefully without repeated upsell attempts.

# Validation
- Run unit tests for upsell candidate selection and stage transitions.
- Manually simulate a completed viewing flow with and without a valid upsell candidate.

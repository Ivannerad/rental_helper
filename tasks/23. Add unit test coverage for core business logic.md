# Why
The highest-risk business rules need fast feedback before integration testing begins.

# Context
- Lives in `tests/` for chatbot policy, models, and shared infrastructure.
- Current behavior: no test suite exists.
- Expected behavior: unit tests cover state transitions, extraction normalization, search logic, and outbound action selection.

# Constraints
- Focus unit tests on deterministic logic, not external network behavior.
- Keep fixtures readable and close to the domain language.
- Avoid brittle test data that obscures why a scenario matters.

# Acceptance Criteria
- Unit tests exist for conversation stages, requirement normalization, listing search, rejection loops, and action generation.
- Test fixtures cover Russian-language user message examples.
- The unit test suite can run without live Telegram, RabbitMQ, or LLM dependencies.

# Validation
- Run the full unit test command.
- Manually review that the covered scenarios map to the implementation plan.

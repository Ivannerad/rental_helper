# Why
Create the base Python monorepo structure so implementation can start without ad hoc file placement.

# Context
- Lives in the repository root and initial `src/`, `tests/`, and service package directories.
- Current behavior: the repo contains only planning documents.
- Expected behavior: the repo has a predictable layout for `admin_bot`, `userbot_worker`, `chatbot_service`, and shared packages.

# Constraints
- Keep importable code out of the repository root.
- Do not lock the project into a framework that conflicts with Telethon or RabbitMQ usage.
- Keep the structure simple enough for local development and containerization later.

# Acceptance Criteria
- `src/` exists with `admin_bot/`, `userbot_worker/`, `chatbot_service/`, and shared package directories.
- `tests/` exists with matching test package placeholders.
- The package layout is documented in the README or service-level docs.

# Validation
- Run the chosen test discovery command to confirm package imports work.
- Manually verify the directory tree matches the planned architecture.

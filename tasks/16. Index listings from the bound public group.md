# Why
The chatbot needs a local searchable inventory source to offer real listings in v1.

# Context
- Lives across `src/userbot_worker/` or a dedicated ingestion module plus persistence.
- Current behavior: bound group posts are not stored or indexed.
- Expected behavior: relevant group posts are ingested, normalized, and saved as searchable listings tied to the account and group.

# Constraints
- Use the bound public group as the only v1 inventory source.
- Ingestion must support initial backfill and incremental updates.
- Listing extraction should not fabricate structured fields when source data is incomplete.

# Acceptance Criteria
- Recent posts from the bound group can be backfilled into the listing store.
- New relevant group posts are indexed as they appear.
- Listings store price, district, room count, free-text summary, and source message ID where available.

# Validation
- Run tests for listing ingestion and normalization.
- Manually ingest a sample group history and verify stored listing records.

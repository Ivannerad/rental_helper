# Why
The admin interface must be restricted and structured before account management features are added.

# Context
- Lives in `src/admin_bot/`.
- Current behavior: no admin bot exists.
- Expected behavior: an admin-only Telegram bot starts, validates caller IDs, and routes supported commands.

# Constraints
- Only allow configured admin Telegram IDs to execute commands.
- Reject unauthorized requests without leaking sensitive account details.
- Keep command parsing simple and auditable.

# Acceptance Criteria
- The admin bot starts and handles at least a base set of commands.
- Unauthorized users are denied.
- Command routing is structured to support future account and schedule handlers.

# Validation
- Run unit tests for access control and command parsing.
- Manually verify authorized and unauthorized command behavior in Telegram.

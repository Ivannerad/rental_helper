# Why
The chatbot must be able to reply to users or forward group listings through the managed account.

# Context
- Lives in `src/userbot_worker/` and shared outbound event handlers.
- Current behavior: no outbound action consumer exists.
- Expected behavior: the worker consumes chatbot actions and performs Telegram sends or group-message forwards.

# Constraints
- Enforce account enabled status before sending.
- Forwarding must only use the group bound to the target account.
- Failed sends should be retried or dead-lettered according to transport policy.

# Acceptance Criteria
- The worker consumes outbound action events from RabbitMQ.
- `send_text` and `forward_group_message` are both implemented.
- Errors are logged with correlation IDs and handled without crashing the worker.

# Validation
- Run integration tests for outbound action consumption and Telegram side effects.
- Manually publish a test action and confirm the corresponding Telegram result.

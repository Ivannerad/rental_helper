# Why
All three services need a single, explicit way to load environment-specific settings safely.

# Context
- Lives in a shared config package under `src/`.
- Current behavior: no runtime configuration model exists.
- Expected behavior: services load Telegram, PostgreSQL, RabbitMQ, LLM, and admin access settings from typed configuration objects.

# Constraints
- Avoid scattering raw environment variable access across services.
- Configuration must support different values per environment.
- Secrets must not be committed to the repository.

# Acceptance Criteria
- A shared config module exposes typed settings for all required integrations.
- Missing required settings fail fast with clear errors.
- Example environment variables are documented.

# Validation
- Run tests for config parsing and validation.
- Manually start each service with sample environment values and confirm startup succeeds.
